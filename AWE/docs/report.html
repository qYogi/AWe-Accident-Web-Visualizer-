AWE Project Report: Accident Data Visualization Tool

1. Introduction

This report provides a detailed overview of the AWE (Accident Web Visualizer) project. The application is a web-based tool designed for the dynamic and flexible visualization of accident data from the USA.

The primary goal of AWE is to facilitate the exploration, analysis, and export of accident data through an intuitive user interface, offering multiple visualization formats and advanced multi-criteria search capabilities.

2. System Purpose and Functionality

The AWE application allows users to:

  • Visualize accident data from the USA for a selected time interval.
  • Generate statistics and visualizations, including interactive maps, bar charts, and pie charts.
  • Export data tables in CSV format and graphics in various image formats.
  • Perform multi-criteria searches to filter incidents based on specific conditions.

3. Implemented Features

  • Time Interval Selection: Users can select a specific date range to view accident data.
  • Data Visualization: The application generates and displays statistics and visualizations in multiple formats:
    • An interactive map displaying accident locations, leveraging the Leaflet.js library.
    • Bar charts and pie charts for statistical representation, built with Chart.js.
    • A paginated and sortable data table.
  • Export Functionality: Users can export the filtered data table to CSV format and save generated charts as images (CSV, WebP, SVG).
  • Multi-Criteria Search: A comprehensive search form allows filtering by:
    • State and city
    • Date and time of day.
    • Severity level.

4. User Interface and Interaction

  • The interface provides a dedicated panel for selecting search filters, including date pickers, dropdowns, and sliders.
  • Visualizations (Table, Map, Charts) are displayed in the main content area and update dynamically based on the selected filters without requiring a page reload.
  • Export buttons for CSV and graphics are clearly accessible within their respective sections.
  • The application provides visual feedback during data loading and displays a message when no results are found.

5. Technical Specifications

  • The application uses the Leaflet.js library for rendering interactive maps.
  • The interface is responsive, ensuring accessibility from both desktop and mobile devices.
  • Data and graphic exports are provided in standard formats (CSV, WebP, SVG).

6. Glossary

  • Map-based visualization: Displaying accident data on an interactive map using Leaflet.js.
  • CSV Export: Saving the current data table as a comma-separated values text file.
  • Multi-criteria search: Advanced search using simultaneous filters for location, date, severity, and weather.
